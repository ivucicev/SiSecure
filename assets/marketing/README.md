# SiSecure marketing assets

Generated by the /marketing pipeline (marketing-studio engine). All video renders
use real SiSecure product footage (Playwright capture of the actual running app —
onboarding, contact requests, safety-number verification, PIN vault, Temp Rooms),
not staged mockups. Copy is grounded in `README.md` and traced to sourced claims;
see the engine repo's `out/sisecure/marketing/brief.json` for the full content brief.

## `matrix/` — responsive aspect exports

Each of `launch` (hero video) and `social` (X/LinkedIn clip) fanned into 4 aspects
by responsive layout (not crops): `16x9` (web/YouTube), `9x16` (Reels/Shorts/TikTok),
`1x1` (feed square), `4x5` (Instagram portrait). `.png` = poster still, `.mp4` = video.

Note: 9:16 and 1:1 muted-autoplay rows normally also get a `-captioned` variant with
burned-in VO captions — skipped this run, see Known gaps below.

## `thumbs/` — poster stills

One JPG per aspect, pulled from the matrix renders.

## `postkit/` — paste-ready per-platform folders

`x/`, `linkedin/`, `youtube/` each contain the right-aspect video, a thumbnail,
`caption.txt` (paste as post copy), `alt.txt` (accessibility description), and a
`POST.md` checklist with platform-specific upload notes. `manifest.json` indexes
all platforms including the two below that are thumb-only this run.

## Known gaps (both from the same environment limitation, not a content issue)

- **No music/voiceover.** The `ELEVENLABS_API_KEY` in the engine repo's `.env` is
  being rejected with a 401 (invalid, not just missing) — all videos ship silent,
  which is the engine's documented fallback deliverable. Fix the key and re-run
  `node scripts/build-sisecure-audio.mjs && node scripts/merge-launch-audio.mjs sisecure`
  in the marketing-studio repo to add a scored soundtrack.
- **No TikTok/Shorts/Instagram video in the postkit** (those `postkit/` folders
  have a thumbnail only) and **no burned-caption 9:16/1:1 matrix variants** — both
  require the audio pass above (captions are derived from VO timing).
- **No standalone 3D logo-reveal video.** Blender isn't installed on this machine.
  The mark still appears everywhere else (launch video, OG assets, social clips)
  via the 2D Remotion fallback — only the dedicated Blender draw-on animation is
  missing. Set `BLENDER_PATH` in the engine's `.env` and run `/logo-reveal` to add it.

## Hook A/B

Three headline takes were rendered and registered as Mission Control variants
(contrarian / value / story categories) — the picture-locked `launch.mp4` above
uses the contrarian take. See `out/sisecure/marketing/hooks/picker.html` in the
engine repo to compare, or re-lock with a different winner via
`scripts/render-hook-variants.mjs`.
